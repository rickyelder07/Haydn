Create `src/audio/midi/` directory and `MidiInputEngine.ts` as a singleton class following the project's AudioEngine pattern (module-level `let instance: MidiInputEngine | null = null` + `getMidiInputEngine()` export).

The class must:
1. `isWebMidiSupported(): boolean` — guard: `typeof window !== 'undefined' && typeof navigator !== 'undefined' && typeof navigator.requestMIDIAccess !== 'undefined'`
2. `async requestAccess(): Promise<void>` — calls `navigator.requestMIDIAccess({ sysex: false })`, attaches input listeners, attaches `onstatechange` for hot-plug/unplug. Throws typed errors: unsupported browser → "Web MIDI API is not supported. Please use Chrome, Edge, or Opera.", NotAllowedError → "MIDI access denied. Please allow MIDI access in your browser settings.", other errors re-thrown.
3. `private attachInputListeners(): void` — iterates `midiAccess.inputs.forEach()`, sets `input.onmidimessage` to call all registered handlers.
4. `private handleStateChange(event: MIDIConnectionEvent): void` — calls `attachInputListeners()` to re-attach after hot-plug, then calls `getMidiInputStore().getState().refreshDevices(this.midiAccess)` to update device list.
5. `getInputDevices(): string[]` — returns `Array.from(midiAccess.inputs.values()).map(i => i.name || 'Unknown Device')`.
6. `addMessageHandler(handler: (event: MIDIMessageEvent) => void): () => void` — adds to `Set<handler>`, returns cleanup function that removes it.
7. `dispose(): void` — sets all input.onmidimessage to null, clears midiAccess.

Define inline TypeScript interfaces (do NOT install @types/webmidi — conflicts with tsconfig lib). Declare minimal interfaces:
```typescript
interface MIDIAccess {
  inputs: Map<string, MIDIInput>;
  onstatechange: ((event: Event) => void) | null;
}
interface MIDIInput {
  name?: string;
  onmidimessage: ((event: MIDIMessageEvent) => void) | null;
}
interface MIDIConnectionEvent extends Event {
  port: { type: string; state: string; name?: string };
}
interface MIDIMessageEvent extends Event {
  data: Uint8Array;
  timeStamp: number;
}
```

Export `getMidiInputEngine(): MidiInputEngine` as the sole entry point. Components never call navigator.requestMIDIAccess directly.

Note: The circular import between MidiInputEngine → getMidiInputStore and midiInputStore → getMidiInputEngine is resolved by late binding — MidiInputEngine only calls getMidiInputStore() inside the onstatechange callback (not at module load time).

**Part B — Create `src/state/midiInputStore.ts`:**

Follow the established Zustand store pattern (playbackStore.ts). Define this interface:
```typescript
interface MidiInputState {
  // Connection
  isSupported: boolean | null;   // null = not checked yet
  isConnected: boolean;
  devices: string[];             // device names for display
  error: string | null;

  // Recording
  isArmed: boolean;
  isRecording: boolean;
  recordingStartTicks: number;
  pendingNotes: Map<number, { startTicks: number; velocity: number }>;
  sustainActive: boolean;
  sustainedNoteOffs: Set<number>;

  // Actions
  connect: () => Promise<void>;
  disconnect: () => void;
  refreshDevices: (access: MIDIAccess | null) => void;
  setArmed: (armed: boolean) => void;
  startRecording: (startTicks: number) => void;
  stopRecording: () => void;
  setIsRecording: (recording: boolean) => void;
  setError: (error: string | null) => void;
}
```

Implement actions:
- `connect()`: check `getMidiInputEngine().isWebMidiSupported()`, set `isSupported: false` + error if not. Call `getMidiInputEngine().requestAccess()` in try/catch — on success set `isConnected: true, devices: getMidiInputEngine().getInputDevices(), error: null, isSupported: true`. On catch set `error: err.message`.
- `disconnect()`: set `isConnected: false, devices: [], isArmed: false, isRecording: false`.
- `refreshDevices(access)`: re-read device names from `getMidiInputEngine().getInputDevices()`, set `devices`.
- `setArmed(armed)`: set `isArmed: armed`. If unArming while recording, also set `isRecording: false`.
- `startRecording(startTicks)`: set `isRecording: true, recordingStartTicks: startTicks, pendingNotes: new Map(), sustainedNoteOffs: new Set()`.
- `stopRecording()`: set `isRecording: false`.
- `setIsRecording` / `setError`: simple setters.

Export `useMidiInputStore` (for React hooks) and `getMidiInputStore` (for singleton access outside React):
```typescript
export const useMidiInputStore = create<MidiInputState>(...);
export const getMidiInputStore = () => useMidiInputStore;
```

Also re-export the MIDIAccess inline interface for use in MidiInputEngine. Place the inline MIDI type declarations in a shared location or re-declare in each file — the simplest approach is to declare them in midiInputStore.ts and import in MidiInputEngine.ts (avoids circular dep since types are erased at runtime).
