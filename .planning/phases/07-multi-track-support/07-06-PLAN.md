  <what-built>
Complete multi-track support system:
- Track UI store with mute/solo/order state
- NoteScheduler track mute via Tone.Part.mute
- Draggable track list with reordering
- Mute/solo buttons per track
- Add track modal with instrument selection
- Remove track with confirmation
- Ghost notes in piano roll with per-track colors
- Ghost notes visibility toggle
  </what-built>
  <how-to-verify>
Start dev server: `npm run dev` and open http://localhost:3000

**Test 1: Track Creation (TRACK-01, TRACK-02)**
1. Generate or upload a MIDI project
2. Click "Add Track" button
3. Select an instrument (e.g., Electric Bass)
4. Enter optional name or leave blank
5. Click Add - verify new empty track appears
6. Repeat until you have 5+ tracks
7. Verify track count shows "X/32"
8. Expected: Can create up to 32 tracks, Add button disabled at limit

**Test 2: Track Removal (TRACK-03)**
1. Click the X/delete button on a track
2. Verify confirmation dialog appears
3. Click Cancel - track should remain
4. Click X again, confirm deletion
5. Expected: Track is removed from list
6. Try to remove last remaining track - should be prevented

**Test 3: Solo Functionality (TRACK-04)**
1. With multiple tracks, press Play
2. Click Solo (S) button on one track
3. Expected: Only that track is audible
4. Click Solo on a second track
5. Expected: Both soloed tracks play (non-exclusive solo)
6. Click Solo again on both to unsolo
7. Expected: All tracks play again

**Test 4: Mute Functionality (TRACK-05)**
1. During playback, click Mute (M) on one track
2. Expected: That track is silent, others play
3. Mute additional tracks
4. Expected: All muted tracks silent
5. Unmute tracks
6. Expected: Audio returns immediately

**Test 5: Mute + Solo Interaction**
1. Solo one track
2. Mute that same track
3. Expected: Track still plays (solo overrides mute in terms of audibility, but mute state is visual)
4. OR: Track is silent (mute takes precedence)
5. Click "Reset Mute/Solo" button
6. Expected: All mute/solo cleared, all tracks audible

**Test 6: Track Reordering**
1. Drag a track by its handle
2. Drop it in a new position
3. Expected: Track list reorders visually
4. Verify playback still works correctly (audio unchanged by display order)

**Test 7: Ghost Notes (Piano Roll)**
1. Select a track in the list
2. Look at piano roll - verify selected track notes are full opacity
3. With ghost notes enabled (toggle should be on by default)
4. Expected: Other tracks visible as semi-transparent colored notes
5. Click toggle to hide ghost notes
6. Expected: Only selected track notes visible
7. Click a ghost note
8. Expected: Nothing happens (no selection, no track switch)

**Test 8: Performance (TRACK-06, 5,000 notes)**
1. Generate a complex project with multiple tracks
2. Or import a large MIDI file
3. Play the project
4. Expected: Smooth playback, no audio glitches
5. Scroll/zoom piano roll with ghost notes enabled
6. Expected: Responsive UI, no significant frame drops

**Test 9: Instrument Display (TRACK-07)**
1. View track list
2. Each track should show:
   - Track name
   - Instrument name below
   - Mute/solo buttons
   - Note count and duration
3. Expected: Clear, readable display of all info

**Test 10: Production Build**
1. Run `npm run build`
2. Expected: Build completes without errors
