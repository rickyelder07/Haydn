Create a new file with WCAG-compliant track colors for multi-track display.

Define:
- `TRACK_COLORS`: readonly array of 12 hex colors with good contrast on dark background (#1a1a1a piano roll)

2. Distinguishable from each other (including for colorblindness)

<task type="auto">
  <name>Task 3: Create trackUIStore</name>
  <files>src/state/trackUIStore.ts</files>
  <action>
Create Zustand store for track-level UI state following existing store patterns (see projectStore.ts, editStore.ts).

Interface TrackUIState:
- `mutedTracks: Set<number>` - Track indices that are muted
- `soloedTracks: Set<number>` - Track indices that are soloed
- `trackOrder: number[]` - Display order (indices into project.tracks)
- `ghostNotesVisible: boolean` - Whether to show non-selected tracks in piano roll

Actions:
- `toggleMute(trackIndex: number)` - Toggle mute for a track
- `toggleSolo(trackIndex: number)` - Toggle solo for a track (non-exclusive)
- `reorderTracks(fromIndex: number, toIndex: number)` - Move track in display order
- `setGhostNotesVisible(visible: boolean)` - Toggle ghost notes
- `resetAllStates()` - Clear all mute/solo (global reset button)
- `initializeOrder(trackCount: number)` - Initialize order array when project loads
- `isTrackAudible(trackIndex: number): boolean` - Computed: track should play audio

Solo/mute logic (per 07-RESEARCH.md):
- If ANY track is soloed: only soloed tracks are audible (regardless of mute)
- If NO tracks are soloed: only non-muted tracks are audible

Export: `useTrackUIStore`
  </action>
