  <action>
    1. Install dependencies: `npm install openai zod js-tiktoken sonner`

    2. Create `src/lib/openai/client.ts`:
       - Import OpenAI from 'openai'
       - Export a singleton OpenAI client that reads OPENAI_API_KEY from process.env
       - Set timeout to 30000ms, maxRetries to 0 (we use custom retry logic)
       - This file runs server-side only (Next.js API route)

    3. Create `src/lib/openai/schemas.ts`:
       - Import z from 'zod'
       - Define individual operation schemas as z.object (NOT discriminated union at root - Structured Outputs limitation):
         - AddNotesParams: { notes: array of {midi: 0-127 int, ticks: non-negative number, durationTicks: positive number, velocity: 0-1 number} }
         - RemoveNotesParams: { noteIndices: array of non-negative ints } (indices of notes to remove from current track)
         - ModifyNotesParams: { modifications: array of {noteIndex: int, updates: {midi?: number, ticks?: number, durationTicks?: number, velocity?: number}} }
         - TransposeParams: { semitones: int between -48 and 48, startTick?: number, endTick?: number }
         - ChangeTempoParams: { newTempo: number between 20 and 300 }
         - ChangeKeyParams: { newKey: string, newScale: enum of 'major' or 'minor' }
       - Define EditOperationSchema as z.object with:
         - operation: z.enum(['add_notes', 'remove_notes', 'modify_notes', 'transpose', 'change_tempo', 'change_key'])
         - parameters: z.union of all param schemas (use z.object with all fields optional, let operation type determine which are present)
         - explanation: z.string() describing what this operation does
       - IMPORTANT: Due to Structured Outputs constraints, avoid discriminated unions at root. Use a flat parameters object approach:
         - parameters: z.object with ALL possible fields as optional (notes?, noteIndices?, modifications?, semitones?, startTick?, endTick?, newTempo?, newKey?, newScale?)
         - This avoids the anyOf limitation while keeping schema valid
       - Define EditResponseSchema as z.object with:
         - operations: z.array(EditOperationSchema)
         - summary: z.string() (overall description of what changed)
         - warnings: z.array(z.string()) (theory violations or concerns)
       - Export all schemas and inferred types (EditResponse, EditOperation)

    4. Create `src/lib/openai/retry.ts`:
       - Define RetryConfig interface: { maxRetries: number, initialDelay: number, maxDelay: number, exponentialBase: number, jitterFactor: number }
       - Export DEFAULT_RETRY_CONFIG: { maxRetries: 3, initialDelay: 2000, maxDelay: 30000, exponentialBase: 2, jitterFactor: 0.3 }
       - Define RETRYABLE_STATUS_CODES: [429, 500, 502, 503, 504]
       - Export async function withExponentialBackoff<T>(fn: () => Promise<T>, config?: RetryConfig): Promise<T>
         - Loop through attempts, catch errors, check if status is retryable
         - Calculate delay: min(initialDelay * base^attempt, maxDelay) + random jitter
         - Use setTimeout promise for delay
         - On final attempt or non-retryable error, throw
       - Export sleep helper function

    5. Create `.env.local` with placeholder: OPENAI_API_KEY=your-api-key-here
  </action>
