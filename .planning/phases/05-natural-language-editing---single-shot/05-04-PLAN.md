<task type="checkpoint:human-verify" gate="blocking">
  <what-built>
    Complete natural language MIDI editing pipeline:
    - CommandInput component above piano roll for typing prompts
    - GPT-4o integration via /api/nl-edit route with Structured Outputs
    - Edit executor applying 6 operation types (add, remove, modify, transpose, tempo, key)
    - Token counting and cost display
    - Undo integration (single step)
    - Retry logic for API failures
    - Info tooltip with example prompts
  </what-built>
  <how-to-verify>
    Prerequisites:
    - Ensure OPENAI_API_KEY is set in .env.local with a valid API key
    - Run `npm run dev` and open http://localhost:3000

    Test 1: Melody editing (Success Criterion 1)
    1. Upload a MIDI file (use im-not-the-only-one---sam-smith.mid or any file with notes)
    2. Select a track in the track list
    3. In the command input above the piano roll, type: "make the melody happier"
    4. Press Enter or click Send
    5. Verify: loading state appears, then notes change in piano roll
    6. Verify: success feedback shows summary + token cost (e.g., "450 tokens (~$0.003)")
    7. Verify: feedback auto-dismisses after ~3.5 seconds

    Test 2: Tempo change (Success Criterion 2)
    1. Type: "speed up to 140 BPM"
    2. Submit and verify tempo changes (check TransportControls tempo display)
    3. Play the track - it should play at the new tempo

    Test 3: Track targeting (Success Criterion 3)
    1. Type: "transpose up an octave" (should apply to currently selected track)
    2. Verify notes in piano roll shifted up 12 semitones

    Test 4: Structured output verification (Success Criterion 4)
    1. Open browser DevTools Network tab
    2. Submit any edit prompt
    3. Check the /api/nl-edit response: should contain { result: { operations: [...], summary: "...", warnings: [...] }, usage: {...} }
    4. Verify operations have correct structure (operation type + parameters + explanation)

    Test 5: Token/cost display (Success Criterion 5)
    1. Submit any edit
    2. Verify success feedback includes both token count and cost: "X tokens (~$Y.ZZZ)"

    Test 6: Undo integration
    1. After any NL edit, press Cmd+Z (Mac) or Ctrl+Z
    2. Verify the edit is fully reversed in one step (not note-by-note)

    Test 7: Error handling
    1. Temporarily break the API key in .env.local (add extra chars)
    2. Submit a prompt
    3. Verify error message appears and persists until dismissed
    4. Restore correct API key

    Test 8: Info tooltip
    1. Hover over the (i) icon next to the input
    2. Verify tooltip shows example prompts
  </how-to-verify>
