  <action>
    1. Create `src/lib/nl-edit/contextBuilder.ts`:
       - Import HaydnProject, HaydnTrack, HaydnNote from '@/lib/midi/types'
       - Import useValidationStore type info
       - Export MIDIContext interface matching the API route's expected input:
         ```
         currentTrack: { index: number, name: string, notes: Array<{midi, ticks, durationTicks, velocity}>, instrument: string }
         otherTracks: Array<{ index: number, name: string, noteCount: number, midiRange: [number, number], instrument: string }>
         project: { tempo: number, timeSignature: string, ppq: number }
         theoryValidation?: { enabled: boolean, currentKey: string, currentGenre: string }
         ```
       - Export function buildCompactMIDIContext(project: HaydnProject, selectedTrackIndex: number, theoryState: {enabled, currentScaleName, activeGenre}): MIDIContext
         - Current track: include ALL note data (midi, ticks, durationTicks, velocity) - no conversion needed, already in correct format
         - Other tracks: only include name, noteCount, midiRange (min/max midi), instrument - filter tracks with notes.length > 0
         - Project: first tempo BPM, first time sig as "N/D" string, ppq
         - Theory: only include if enabled, with currentScaleName and activeGenre
         - Handle edge cases: empty tracks (skip in otherTracks), missing tempo (default 120), missing time sig (default "4/4")

    2. Create `src/lib/nl-edit/editExecutor.ts`:
       - Import types from '@/lib/midi/types' and '@/lib/openai/schemas' (EditResponse, EditOperation)
       - Import useProjectStore and useEditStore
       - Export interface EditExecutionResult: { success: boolean, changedNoteIndices: number[], summary: string, error?: string }
       - Export function executeEditOperations(operations: EditOperation[], trackIndex: number): EditExecutionResult
         - Get current project from useProjectStore.getState()
         - Get editStore state for history management
         - Process each operation in order:
           - 'add_notes': Add notes to the specified track (use parameters.notes array), push midi/ticks/durationTicks/velocity. Track which indices are new (changedNoteIndices).
           - 'remove_notes': Remove notes by index from parameters.noteIndices. Sort indices descending before splicing to maintain correct positions. Track removed count.
           - 'modify_notes': For each modification in parameters.modifications, update the note at noteIndex with the provided updates (midi, ticks, durationTicks, velocity). Track modified indices.
           - 'transpose': Shift all notes in the track (or range startTick-endTick if specified) by parameters.semitones. Clamp resulting midi to 0-127. Track all transposed indices.
           - 'change_tempo': Update project.metadata.tempos[0].bpm to parameters.newTempo. If no tempos exist, create one at ticks:0.
           - 'change_key': Update project.metadata.keySignatures. If keySignatures exist, update first entry. If empty, add new entry at ticks:0 with parameters.newKey and parameters.newScale.
         - After all operations:
           - Sort notes by ticks (maintain chronological order)
           - Push new notes state to editStore history (for undo support) by calling the editStore's _history.push()
           - Sync updated project to projectStore via setProject()
           - Return result with changedNoteIndices and combined summary
         - IMPORTANT for undo integration: The executor should NOT use editStore.addNote/deleteNote/moveNote individually (those each create separate history entries). Instead, it should:
           1. Clone current track notes
           2. Apply ALL operations to the clone
           3. Push the final result as ONE history entry (single undo step)
           4. Sync to projectStore
         - Wrap in try/catch, return { success: false, error: message } on failure
  </action>

    1. Create `src/state/nlEditStore.ts`:
       - Import create from 'zustand'
       - Import useProjectStore from './projectStore'
       - Import useEditStore from './editStore'
       - Import useValidationStore from './validationStore'
       - Import buildCompactMIDIContext from '@/lib/nl-edit/contextBuilder'
       - Import executeEditOperations from '@/lib/nl-edit/editExecutor'
       - Import estimateTokensAndCost, calculateActualCost, formatCost from '@/lib/openai/tokenCounter'
       - Import EditResponse type from '@/lib/openai/schemas'

       - Define NLEditState interface:
         - status: 'idle' | 'estimating' | 'submitting' | 'executing' | 'success' | 'error'
         - currentPrompt: string
         - tokenEstimate: TokenEstimate | null
         - error: string | null
         - lastEditResult: { summary: string, costDisplay: string, changedNoteIndices: number[] } | null
         - Actions: setPrompt, submitEdit, reset, dismissError, dismissSuccess

       - Implement store:
         - setPrompt(prompt: string): set currentPrompt and clear error
         - submitEdit(): the main async action:
           1. Get currentPrompt, validate non-empty
           2. Set status: 'estimating'
           3. Build MIDI context using buildCompactMIDIContext with data from projectStore, editStore, validationStore
           4. Estimate tokens using estimateTokensAndCost (build a simplified system prompt for estimation)
           5. If isExpensive, set status to 'idle' and set error to cost warning message (UI will show confirmation) - for v1, just block with error message rather than modal
           6. Set status: 'submitting'
           7. fetch POST /api/nl-edit with { prompt, context }
           8. If response not ok, throw with error message from response
           9. Parse response JSON: { result: EditResponse, usage: { promptTokens, completionTokens, totalTokens } }
           10. Set status: 'executing'
           11. Call executeEditOperations(result.operations, editStore.selectedTrackIndex)
           12. Calculate actual cost from usage stats
           13. Set status: 'success', populate lastEditResult with summary, costDisplay, changedNoteIndices
           14. Auto-clear success after 3500ms: setTimeout(() => set({ status: 'idle', lastEditResult: null }), 3500)
           15. Clear currentPrompt on success
           Catch: set status: 'error', set error message
         - reset(): reset all state to initial values
         - dismissError(): set error to null, status to idle
         - dismissSuccess(): clear lastEditResult, set status to idle

    2. Add a batch edit method to `src/state/editStore.ts`:
       - Add new action: `applyBatchEdit: (notes: HaydnNote[]) => void`
       - This replaces the current track's notes with the provided array in one history step
       - Implementation: same as existing addNote but takes complete notes array, pushes to history, syncs to projectStore
       - This is needed so editExecutor can apply multiple operations as a single undoable action
       - Keep all existing actions unchanged
  </action>
