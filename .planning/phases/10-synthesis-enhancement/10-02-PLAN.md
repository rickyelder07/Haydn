  <action>
    Modify `src/audio/playback/NoteScheduler.ts` to pre-load all unique instruments in parallel before the scheduling loop.

    Current state: instruments are loaded sequentially inside a `for` loop (one `await createInstrument()` per track, one at a time). With CDN loading (~200-400ms per instrument), this waterfall is unacceptably slow for projects with multiple unique instruments.

    Required change — restructure `scheduleNotes()`:

    1. Before the existing `for` loop that creates Parts, add a parallel pre-load phase:
       ```
       // Collect unique instrument keys that need loading
       const toLoad: Array<{ key: number | string; gmProgram: number; channel: number }> = [];
       for (const track of project.tracks) {
         if (track.notes.length === 0) continue;
         const isPercussion = isPercussionChannel(track.channel);
         const instrumentKey = isPercussion ? 'percussion' : track.instrumentNumber;
         if (!loadedInstruments.has(instrumentKey) && !toLoad.some(x => x.key === instrumentKey)) {
           toLoad.push({ key: instrumentKey, gmProgram: track.instrumentNumber, channel: track.channel });
         }
       }
       // Load all new instruments in parallel
       await Promise.all(
         toLoad.map(async ({ key, gmProgram, channel }) => {
           const instrument = await createInstrument(gmProgram, channel);
           loadedInstruments.set(key, instrument);
         })
       );
       ```

    2. In the existing scheduling loop that creates Parts, remove the `await createInstrument()` call and `loadedInstruments.set()` call — instruments are now pre-loaded. Just read from `loadedInstruments.get(instrumentKey)`.

    3. Update the `instrumentKey` logic to match the pre-load phase: `isPercussion ? 'percussion' : track.instrumentNumber`. Remove the old `isPiano` special-casing (programs 0-7 grouping under key 0) — with real samples every program is its own instrument. Each unique program number is its own key.

    4. The `soundfontLoader.ts` fetch cache handles de-duplication at the network layer — if two tracks share the same program, `loadSoundfontNotes()` returns the same cached Promise for both, so they don't double-fetch.

    Keep all other existing functions (`clearScheduledNotes`, `disposeAllInstruments`, `setTrackMuted`, `updateTrackMuteStates`, `getLoadedInstrumentCount`) unchanged.
  </action>
