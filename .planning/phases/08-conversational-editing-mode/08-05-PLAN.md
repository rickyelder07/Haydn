  <how-to-verify>
**Setup:**
1. Run `npm run dev`
2. Open http://localhost:3000
3. Load a MIDI file or generate a new project (e.g., "Create a simple pop beat")
4. Select a track
5. Switch to "Conversation" mode using the toggle

**Test Phase Success Criteria:**

**Criterion 1: Multiple edits in sequence**
- Type: "add some drums"
- Wait for response
- Type: "make them louder"
- Wait for response
- Type: "add swing to the hi-hats"
- PASS if: Each edit builds on previous, conversation history shows all turns

**Criterion 2: Preserved context**
- Start new conversation
- Type: "add a bass line"
- Type: "make it funkier"
- PASS if: "it" correctly resolves to bass line, not other elements

**Criterion 3: Structural changes**
- Type: "add a bridge section" or "make the chorus 8 bars longer"
- PASS if: Structure modification applies (may add notes, change sections)

**Criterion 4: Note-level changes**
- Type: "raise the melody an octave" or "change to minor key"
- PASS if: Notes transpose correctly or key signature updates

**Criterion 5: Instrument swapping**
- Type: "replace piano with strings" or "add drums"
- PASS if: Instrument changes apply or track adds

**Criterion 6: Timing adjustments**
- Type: "speed up to 140 BPM" or "swing the hi-hats"
- PASS if: Tempo changes or note timing adjusts

**Additional checks:**
- Token costs accumulate and display correctly
- New/Clear buttons work
- Error handling shows failures gracefully
- Single-shot mode still works when switched back
