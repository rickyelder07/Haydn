Create Zustand store following existing nlEditStore.ts patterns:

1. **State interface (ConversationState):**
   - activeSession: ConversationSession | null
   - isLoading: boolean
   - error: string | null

2. **Actions interface (ConversationActions):**
   - startNewSession(): void - Creates new session with UUID, timestamp, empty turns, "New Conversation" title
   - clearSession(): void - Sets activeSession to null, clears error
   - addTurn(turn: ConversationTurn): void - Appends turn to activeSession.turns, updates lastUpdatedAt, sets title from first prompt if turns.length was 0
   - setLoading(loading: boolean): void
   - setError(error: string | null): void
   - getTotalTokens(): number - Sum of all turns' tokenUsage.totalTokens
   - getTotalCost(): number - Sum of all turns' tokenUsage.estimatedCost

3. **Store implementation:**
   - Use `create<ConversationState & ConversationActions>()` pattern
   - NO persist middleware for now (session only, per research recommendation)
   - startNewSession uses crypto.randomUUID() for ID generation
   - Title auto-generation: truncate first prompt to 50 chars with "..." if longer

Follow the exact Zustand patterns from nlEditStore.ts. Do NOT use persist middleware - research says start with session persistence only.
