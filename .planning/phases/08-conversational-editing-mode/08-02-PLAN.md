<task type="auto">
  <name>Task 2: Create conversation API route</name>
  <files>src/app/api/nl-conversation/route.ts</files>
  <action>
Create POST route at /api/nl-conversation following existing nl-edit/route.ts pattern:

1. **Request body interface:**
   ```typescript
   {
     prompt: string;
     conversationHistory: Array<{ role: 'user' | 'assistant'; content: string }>;
     context: MIDIContext;
   }
   ```

2. **Validation:**
   - Prompt required, non-empty, max 500 chars (same as single-shot)
   - Context required and must be object
   - conversationHistory optional (defaults to empty array for first turn)

3. **Message building:**
   - Build system prompt with buildSystemPromptForConversation(context)
   - Start messages array with system prompt
   - Add conversationHistory if provided
   - Add new user prompt

4. **GPT-4o call:**
   - Use withExponentialBackoff (from existing retry.ts)
   - Use openai.chat.completions.parse with EditResponseSchema
   - Model: 'gpt-4o-2024-08-06'

5. **Response:**
   ```typescript
   {
     result: EditResponse;
     usage: { promptTokens, completionTokens, totalTokens };
     // Note: Edit execution happens client-side in the store
   }
   ```

6. **Error handling:**
   - 400 for invalid request
   - 500 for OpenAI errors
   - Log errors with console.error

The key difference from nl-edit is accepting conversationHistory and including it in the messages array. Edit execution happens client-side (store calls editExecutor after API response).
  </action>
