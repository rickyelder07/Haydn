**midiAssembler.ts** - Assemble generator outputs into a complete HaydnProject:

```
function assembleProject(params: GenerationParams): HaydnProject
```

Import GenerationParams type from `@/lib/openai/schemas`.
Import all generators: generateChordProgression, generateDrumTrack, generateMelodyTrack, generateBassTrack.
Import HaydnProject, HaydnTrack, HaydnNote from `@/lib/midi/types`.

Implementation:
1. Set PPQ = 480 (standard MIDI resolution, matches existing codebase)
2. Generate chord progression first (other generators depend on it):
   ```
   const { notes: chordNotes, chordSymbols } = generateChordProgression(
     params.key, params.scale, params.structure, params.genre, PPQ
   )
   ```
3. Generate drum track: `generateDrumTrack(params.structure, params.genre, PPQ, params.emotion.arousal)`
4. Generate melody: `generateMelodyTrack(chordSymbols, params.key, params.scale, params.genre, PPQ, params.emotion)`
5. Generate bass: `generateBassTrack(chordSymbols, params.key, params.scale, params.genre, PPQ)`
6. Build HaydnTrack objects for each role using instrumentation from params:
   - Find the instrument number for each role from params.instrumentation array
   - Fall back to GENRE_TEMPLATES[genre].defaultInstrumentation if role not in params
   - Use GM_INSTRUMENT_NAMES from `@/lib/midi/gmInstruments` (or define inline) for instrument names
   - Drums track: channel 9 (GM percussion), all other tracks: channels 0, 1, 2
   - Track names: "{Genre} Drums", "{Genre} Bass", "{Genre} Chords", "{Genre} Melody"
7. Calculate total duration in ticks: sum of all section bars * 4 beats * PPQ
8. Assemble HaydnProject:
   ```
   {
     originalFileName: null,
     sourceFormat: 'new',
     metadata: {
       name: params.description || 'Generated Track',
       ppq: PPQ,
       tempos: [{ ticks: 0, bpm: params.tempo }],
       timeSignatures: [{ ticks: 0, numerator: params.timeSignatureNumerator, denominator: params.timeSignatureDenominator }],
       keySignatures: [{ ticks: 0, key: params.key, scale: params.scale === 'major' ? 'major' : 'minor' }]
     },
     tracks: [drumTrack, bassTrack, chordTrack, melodyTrack].filter(t => t.notes.length > 0),
     durationTicks: totalDurationTicks
   }
   ```
9. Filter out empty tracks (e.g., classical has no drums)

For instrument name lookup: Import GM_INSTRUMENT_NAMES from `@/lib/midi/gmInstruments` if it exists. If not, define a simple lookup inline: `const getGMName = (program: number) => 'Instrument ' + program;` and note that the actual GM names map exists at `@/lib/midi/gmInstruments.ts` from Phase 1.

**nlGenerationStore.ts** - Zustand store following same pattern as nlEditStore.ts:

State interface:
```typescript
interface NLGenerationState {
  isGenerating: boolean;
  lastPrompt: string | null;
  lastError: string | null;
  tokenUsage: { promptTokens: number; completionTokens: number; totalTokens: number; estimatedCost: number } | null;
}
```

Actions:
```typescript
interface NLGenerationActions {
  submitGeneration: (prompt: string) => Promise<void>;
  clearError: () => void;
  reset: () => void;
}
```

submitGeneration implementation:
1. Set `{ isGenerating: true, lastError: null }`
2. POST to `/api/nl-generate` with `{ prompt }` body
3. On success: parse response as `{ params: GenerationParams, usage: {...} }`
4. Call `assembleProject(params)` to generate the HaydnProject
5. Call `useProjectStore.getState().setProject(project)` to load into the app
6. Reset editStore selected track: `useEditStore.getState().selectTrack(0)` (select first track after generation - but only if tracks exist)
7. Calculate estimated cost using same formula as nlEditStore ($2.50/M input, $10/M output)
8. Set success state: `{ isGenerating: false, lastPrompt: prompt, tokenUsage: { ... } }`
9. On error: set `{ isGenerating: false, lastError: message }`

IMPORTANT: Import projectStore and editStore using `.getState()` pattern (cross-store access, same as editExecutor.ts pattern). Do NOT import React hooks.

**GenerationInput.tsx** - Create a generation prompt input component following the same visual pattern as CommandInput.tsx:

The component should:
1. Use `useNLGenerationStore` for state and actions
2. Have a local `useState` for the input text
3. Submit on Enter or button click (same handleSubmit/handleKeyDown pattern as CommandInput)
4. Show inline feedback:
   - Loading: spinner + "Generating..." text
   - Error: red bg with error message and dismiss button
   - Success: green bg with "Generated! [description]" and token cost
5. Include an info tooltip with example generation prompts:
   - "create a lo-fi hip hop beat"
   - "make a trap beat in C# minor at 150 BPM"
   - "compose a jazz ballad in Bb major"
   - "generate a pop song, 120 BPM, 4/4 time"
   - "create a classical piano piece in D minor"
6. Input placeholder: "Describe what to create... (e.g., 'create a lo-fi beat in D minor')"
7. Button text: "Generate" (or "Generating..." when loading)
8. Max length 500 chars (same as CommandInput)

Visual difference from CommandInput: Use a purple/indigo accent color (bg-indigo-600, border-indigo-500) instead of blue to visually distinguish generation from editing. This helps users understand they're creating new content vs editing existing content.

Reuse the same SVG icon components (InfoIcon, SpinnerIcon, XIcon) - either import from a shared location or define inline (same as CommandInput does).
