**chordGenerator.ts** - Create a function that generates chord voicings as HaydnNote arrays:

```
function generateChordProgression(
  key: string,
  scale: string,
  structure: Array<{ section: string, bars: number }>,
  genre: string,
  ppq: number
): { notes: HaydnNote[], chordSymbols: Array<{ chord: string, startTick: number, durationTicks: number }> }
```

Implementation:
1. For each section in structure, select a chord progression from GENRE_TEMPLATES[genre].chordProgressions[sectionType]
2. Use `Progression.fromRomanNumerals(key, romanNumerals)` from Tonal.js to convert to actual chord names
3. Each chord lasts one bar (4 beats = 4 * ppq ticks) by default
4. Repeat/extend the 4-chord progression to fill the section's bar count
5. For each chord, create HaydnNote entries for a simple voicing:
   - Get chord notes using `Chord.get(chordName).notes` from Tonal.js
   - Place in octave 4 (MIDI 60-72 range) using `Note.midi(noteName + '4')` from Tonal.js
   - Each note starts at the bar's tick offset, duration = 4 * ppq (whole bar)
   - Velocity 0.6 for chords (accompaniment level)
6. Return both the HaydnNote array AND the chord symbols with timing (needed by melody/bass generators)

IMPORTANT: Use `import { Progression, Chord, Note } from 'tonal'` (the main tonal package, not subpackages - decision from Phase 4).

Handle edge cases:
- If Progression.fromRomanNumerals returns empty, fall back to simple I-IV-V-I
- If a chord name can't be parsed, skip it (don't crash)
- Clamp MIDI values to 0-127

**rhythmGenerator.ts** - Generate drum tracks from genre templates:

```
function generateDrumTrack(
  structure: Array<{ section: string, bars: number }>,
  genre: string,
  ppq: number,
  arousal: number // -1 to 1, affects pattern density
): HaydnNote[]
```

Implementation:
1. Get drum patterns from GENRE_TEMPLATES[genre].drumPatterns
2. For each section, select the pattern for that section type (fall back to 'verse' pattern if section not found)
3. For each bar in the section, stamp the pattern:
   - Kick: MIDI 36 (GM Bass Drum), velocity 0.9
   - Snare: MIDI 38 (GM Acoustic Snare), velocity 0.85
   - Hi-hat closed: MIDI 42 (GM Closed Hi-Hat), velocity 0.6
   - Open hi-hat: MIDI 46 (GM Open Hi-Hat), velocity 0.65
4. All drum notes use durationTicks = ppq / 4 (16th note duration)
5. Apply velocity variation from template: `velocity * (1 + (Math.random() - 0.5) * 2 * velocityVariation)`
6. Arousal modifier: If arousal > 0.3, add extra kick hits at 8th-note subdivisions (every other bar). If arousal < -0.3, remove some hi-hat hits (keep only quarter notes).
7. If genre is 'classical', return empty array (no drums).

IMPORTANT: All drum notes go on channel 9 (0-indexed) per GM spec.

**melodyGenerator.ts** - Generate melodic lines over chords:

```
function generateMelodyTrack(
  chordSymbols: Array<{ chord: string, startTick: number, durationTicks: number }>,
  key: string,
  scale: string,
  genre: string,
  ppq: number,
  emotion: { valence: number, arousal: number }
): HaydnNote[]
```

Implementation:
1. Get melodyConfig from GENRE_TEMPLATES[genre].melodyConfig
2. Get scale pitch classes using `Scale.get(key + ' ' + scale).notes` from Tonal.js, then map to MIDI chromas using `Note.chroma()`
3. Determine notes per bar: lerp between noteDensityRange[0] and noteDensityRange[1] based on arousal (higher arousal = more notes)
4. Start on the tonic in the middle of the octave range: `Note.midi(key + octaveRange[0])` or similar
5. For each chord in chordSymbols:
   - Get chord tones using `Chord.get(chordName).notes` -> `Note.chroma()` for each
   - Calculate how many bars this chord spans: durationTicks / (4 * ppq)
   - For each bar, generate notesPerBar notes:
     - Calculate tick position: evenly spaced within the bar
     - Choose next note using weighted random walk:
       - 50% chance: chord tone (strong harmonic connection)
       - 35% chance: scale tone (melodic movement)
       - 15% chance: chromatic passing tone (color)
     - Apply stepwise preference: among candidates (same pitch class in different octaves), choose the one closest to current MIDI note
     - Clamp to octaveRange (e.g., MIDI 60-84 for octave 4-5)
   - Duration: ppq (quarter note) for most notes. For the last note in a phrase, use ppq * 2 (half note) for phrasing
   - Velocity: 0.7 base + random variation 0.1 + valence modifier (happier = slightly louder, +0.05 max)
6. On strong beats (beat 1 and 3, i.e., tick offset 0 and 2*ppq within bar), prefer chord tones (bump chord tone probability to 70%)

Use `import { Scale, Chord, Note } from 'tonal'`.

**bassGenerator.ts** - Generate bass lines following chords:

```
function generateBassTrack(
  chordSymbols: Array<{ chord: string, startTick: number, durationTicks: number }>,
  key: string,
  scale: string,
  genre: string,
  ppq: number
): HaydnNote[]
```

Implementation:
1. Get bassConfig from GENRE_TEMPLATES[genre].bassConfig
2. For each chord in chordSymbols:
   - Get root note using `Chord.get(chordName).tonic` from Tonal.js
   - Convert to MIDI in bass octave: `Note.midi(rootNote + bassConfig.octave)` (e.g., "C2" for octave 2)
   - Based on rhythmStyle:

   **'root-notes'** (lofi, trap, pop):
   - Play root on beat 1 (tick offset 0), duration = full chord duration
   - Velocity 0.8

   **'walking'** (boom-bap, jazz):
   - Beat 1: chord root, duration = ppq (quarter note)
   - Beat 2: chord 3rd (or scale degree 3 above root), duration = ppq
   - Beat 3: chord 5th (or scale degree 5 above root), duration = ppq
   - Beat 4: chromatic approach to next chord root (one semitone below next root), duration = ppq
   - Velocity 0.75 with slight variation

   **'arpeggiated'** (classical):
   - Arpeggiate chord tones across the bar in 8th notes
   - Get chord notes, cycle through them at every 8th note (every ppq/2 ticks)
   - Place in bass octave
   - Velocity 0.7

3. Clamp all MIDI values to 0-127

Use `import { Chord, Note } from 'tonal'`.

All three generators are pure functions: they take parameters and return HaydnNote[]. No side effects, no store access.
