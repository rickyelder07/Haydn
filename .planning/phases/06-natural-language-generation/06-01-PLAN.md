Create the GenerationParams Zod schema and API endpoint that uses GPT-4o to parse natural language generation prompts into structured musical parameters.

Purpose: This is the LLM-powered first step of the hybrid generation architecture. GPT-4o excels at understanding user intent ("create a lo-fi hip hop beat") and mapping it to concrete musical parameters (genre: lofi, tempo: 80, key: Eb, scale: minor, etc.). Algorithms in later plans use these parameters to generate actual MIDI notes.

Output: GenerationParamsSchema added to schemas.ts, POST /api/nl-generate route that returns parsed parameters with token usage.

<task type="auto">
  <name>Task 1: Add GenerationParamsSchema to schemas.ts</name>
  <files>src/lib/openai/schemas.ts</files>
  <action>
Add a new Zod schema `GenerationParamsSchema` to the existing schemas.ts file (append after EditResponseSchema, do not modify existing schemas).

The schema must define:
- `genre`: z.enum(['lofi', 'trap', 'boom-bap', 'jazz', 'classical', 'pop']) - the musical genre
- `tempo`: z.number().int().min(40).max(240) - BPM
- `key`: z.string() - musical key like "C", "D#", "Bb"
- `scale`: z.enum(['major', 'minor', 'harmonic minor', 'dorian', 'mixolydian']) - scale type
- `timeSignatureNumerator`: z.number().int().min(2).max(16)
- `timeSignatureDenominator`: z.number().int().min(2).max(16)
- `structure`: z.array(z.object({ section: z.enum(['intro', 'verse', 'chorus', 'bridge', 'outro']), bars: z.number().int().min(1).max(64) }))
- `emotion`: z.object({ valence: z.number().min(-1).max(1), arousal: z.number().min(-1).max(1) }) - valence: happy(1)/sad(-1), arousal: energetic(1)/calm(-1)
- `instrumentation`: z.array(z.object({ role: z.enum(['drums', 'bass', 'melody', 'chords']), instrument: z.number().int().min(0).max(127) })) - GM instrument numbers per role
- `description`: z.string() - original user prompt for reference

IMPORTANT: All fields should use .describe() for OpenAI Structured Outputs clarity. Use .nullable().optional() pattern where needed for Structured Outputs compatibility (learned from Phase 5 - OpenAI requires nullable for optional fields in strict mode). However, since generation extracts ALL parameters (with defaults), make all fields required - GPT must fill in defaults for unspecified values.

Export the schema as `GenerationParamsSchema` and the inferred type as `GenerationParams`.

  <done>GenerationParamsSchema exported from schemas.ts with all fields defined, GenerationParams type inferred and exported, existing EditResponse schemas untouched.</done>

<task type="auto">
  <name>Task 2: Create POST /api/nl-generate endpoint</name>
  <files>src/app/api/nl-generate/route.ts</files>
  <action>
Create a new Next.js API route at `src/app/api/nl-generate/route.ts` following the same pattern as `src/app/api/nl-edit/route.ts`.

The endpoint should:

1. Accept POST with body `{ prompt: string }` (no MIDI context needed for generation - this is the key difference from editing)

2. Validate prompt: non-empty string, max 500 chars (same limit as editing)

3. Build a system prompt that instructs GPT-4o to extract structured generation parameters. The system prompt should include:
   - Role: "You are a music generation parameter extractor"
   - Instructions: Parse user's prompt, extract parameters, fill in genre-appropriate defaults for unspecified values
   - Genre defaults table:
     - lofi: 70-90 BPM, minor scale, 4/4 time, Eb/Bb/Ab keys common, low arousal, instruments: drums(0), bass(33), chords(4), melody(11)
     - trap: 140-160 BPM, minor scale, 4/4 time, dark keys (C#m, Fm), high arousal, instruments: drums(0), bass(38), chords(90), melody(81)
     - boom-bap: 85-95 BPM, minor scale, 4/4 time, nostalgic feel, instruments: drums(0), bass(33), chords(4), melody(65)
     - jazz: 120-180 BPM, dorian/mixolydian, 4/4 time, instruments: drums(0), bass(32), chords(0), melody(66)
     - classical: 60-120 BPM, major/minor, varies, instruments: no drums, bass(42 cello), chords(0 piano), melody(40 violin)
     - pop: 100-130 BPM, major, 4/4 time, instruments: drums(0), bass(33), chords(0), melody(0)
   - Default structure if not specified: intro(4), verse(8), chorus(8), verse(8), chorus(8), outro(4)
   - Note: "Output ONLY the structured parameters. Do not include explanatory text."

4. Call GPT-4o using `openai.chat.completions.parse()` with `zodResponseFormat(GenerationParamsSchema, 'generation_params')`, wrapped in `withExponentialBackoff()` - same pattern as nl-edit route. Use model `gpt-4o-2024-08-06`.

5. Return JSON response: `{ params: GenerationParams, usage: { promptTokens, completionTokens, totalTokens } }`

6. Error handling: 400 for invalid input, 500 for API errors - same pattern as nl-edit route.

IMPORTANT: The system prompt for generation is much smaller than editing (~500 tokens vs 2000-5000) because there is no existing MIDI context to serialize. This is a significant cost advantage.
