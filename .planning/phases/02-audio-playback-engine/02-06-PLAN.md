<task type="checkpoint:human-verify" gate="blocking">
  <what-built>
Complete Audio Playback Engine with:
- MIDI note playback using Tone.js
- Piano tracks use @tonejs/piano samples
- Non-piano tracks use oscillator synthesis
- Transport controls (play/pause/stop)
- Time display (mm:ss and bars:beats)
- Tempo slider (40-240 BPM)
- Click-to-seek progress bar
- Keyboard shortcuts (Space, Enter, Home)
- Note highlighting during playback
- Metronome with downbeat accent
- Count-in (1-4 bars)
  </what-built>
  <how-to-verify>
**Setup:**
1. Run `npm run dev` in terminal
2. Open http://localhost:3000 in browser
3. Have a MIDI file ready (ideally one with:
   - Multiple tracks
   - At least one piano track (GM program 0-7)
   - At least one non-piano track
   - Duration > 1 minute to test timing)

**Test Sequence:**

**A. Basic Playback (PLAY-01)**
1. Upload a MIDI file
2. Verify TransportControls appears below FileUpload
3. Click the Play button (blue circle)
4. Verify: Audio plays, notes are audible
5. Verify: Piano tracks sound like piano (sampled), other tracks sound synthetic

**B. Pause/Resume (PLAY-02)**
1. While playing, click Pause
2. Verify: Audio stops, button shows Play icon
3. Verify: Time display shows paused position (not 0:00)
4. Click Play again
5. Verify: Audio resumes from paused position, not from start

**C. Stop (PLAY-03)**
1. While playing, click Stop (square icon)
2. Verify: Audio stops
3. Verify: Time display returns to 0:00
4. Verify: Progress bar resets to 0%

**D. Tempo Adjustment (PLAY-04)**
1. Note current tempo display (e.g., 120 BPM)
2. Drag tempo slider right (faster)
3. Verify: Audio plays faster immediately
4. Drag tempo slider left (slower)
5. Verify: Audio plays slower immediately
6. Verify: Tempo stays within 40-240 BPM range

**E. Timing Accuracy (PLAY-05)**
1. Play a track that's at least 3 minutes long
2. Let it play through completely
3. Verify: Notes don't drift out of sync with each other
4. Verify: Time display progresses smoothly
5. Verify: Track ends at correct duration

**F. Keyboard Shortcuts**
1. With track stopped, press Spacebar
2. Verify: Playback starts
3. Press Spacebar again
4. Verify: Playback pauses
5. Press Enter
6. Verify: Returns to start (0:00)
7. Verify: Focus in a text input disables shortcuts

**G. Time Display**
1. Verify: mm:ss format shows (e.g., "1:23")
2. Verify: bars.beats format shows (e.g., "5.2.1")
3. Verify: Both update during playback

**H. Click-to-Seek**
1. Click in the middle of the progress bar
2. Verify: Position jumps to clicked location
3. Verify: Time display updates
4. Press Play
5. Verify: Playback continues from seeked position

**I. Note Highlighting**
1. Play a track
2. Watch the TrackList
3. Verify: Tracks with active notes show green indicator
4. Verify: Note count updates during playback

**J. Metronome**
1. Click the "Click" button (metronome toggle)
2. Play the track
3. Verify: Hear click on each beat
4. Verify: First beat of each bar is higher pitch
5. Toggle off
6. Verify: Clicks stop

**K. Count-In**
1. Stop playback
2. Set Count-in to "2 bars"
3. Press Play
4. Verify: Hear 8 clicks (2 bars x 4 beats)
5. Verify: Music starts after count-in completes

**Expected Results:**
- All tests pass
- No JavaScript errors in console
- UI remains responsive during playback
- TypeScript types are correct (`npx tsc --noEmit` passes)
- Production build works (`npm run build` succeeds)
  </how-to-verify>

  </resume-signal>
