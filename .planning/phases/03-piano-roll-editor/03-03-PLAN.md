  <name>Task 1: Create mouse interaction hook</name>
  <files>src/components/PianoRoll/usePianoRollInteractions.ts</files>
  <action>
Create a custom hook `usePianoRollInteractions` that handles all mouse-based editing interactions for the piano roll.

Parameters:
- `canvasRef: React.RefObject<HTMLCanvasElement>` - Reference to the piano roll canvas
- `notes: HaydnNote[]` - Current notes for hit-testing
- `ppq: number` - For snap grid calculation
- `zoomX: number`, `zoomY: number` - Current zoom levels
- `scrollX: number`, `scrollY: number` - Current scroll offsets

Returns:
- `handleMouseDown: (e: React.MouseEvent) => void`
- `handleMouseMove: (e: React.MouseEvent) => void`
- `handleMouseUp: (e: React.MouseEvent) => void`
- `handleContextMenu: (e: React.MouseEvent) => void`
- `dragState: { isDragging: boolean; noteIndex: number | null; startMidi: number; startTicks: number } | null`

Behavior:

**Left-click on empty space** -> Add note:
1. Convert click (x, y) to (ticks, midi) using gridUtils
2. Snap ticks to grid using snapToGrid
3. Create new HaydnNote: { midi, ticks: snappedTicks, durationTicks: ppq (one beat default), velocity: 0.8 }
4. Call editStore.addNote(note)

**Left-click on existing note** -> Select note:
1. Hit-test: find note whose rectangle contains click point
2. Call editStore.selectNote(noteId)
3. If shift held: do not clear prior selection (future multi-select)

**Right-click on note OR Shift+Left-click on note** -> Delete note:
1. Hit-test to find the note index
2. Call editStore.deleteNote(noteIndex)
3. Prevent context menu (e.preventDefault)

**Left-click-drag on note** -> Move note:
1. On mousedown on note: record starting (ticks, midi) and note index, set dragState
2. On mousemove while dragging: calculate delta ticks and delta midi from mouse movement
3. On mouseup: compute final (newTicks, newMidi) snapped to grid, call editStore.moveNote(noteIndex, newMidi, newTicks)
4. During drag, call editStore.setEditing(true). On mouseup, call editStore.setEditing(false).

**Hit-testing:**
- For each note, compute its rectangle: x = ticksToX(note.ticks), width = ticksToX(note.ticks + note.durationTicks) - x, y = midiToY(note.midi), height = NOTE_HEIGHT
- Check if click point falls within rectangle
- Iterate notes in reverse order (top notes drawn last = highest z-index)

Use `useCallback` for stable handler references. Access editStore via `useEditStore()` hook.

**PianoRollEditor.tsx:**
Create the main assembled component that brings together canvas, sidebar, interactions, and editing controls.

This component:
1. Reads from `useProjectStore` to get the current project's tracks and metadata
2. Reads from `useEditStore` to get selectedTrackIndex, canUndo, canRedo, selectedNoteIds
3. Reads from `usePlaybackStore` to get current playhead position (for rendering playhead line)
4. Manages local state for: scrollX, scrollY, zoomX (default 1.0), zoomY (default 1.0)

Layout (use Tailwind):
```
+------------------------------------------+
| Toolbar: [Undo] [Redo] | Mode indicator  |
+------+-----------------------------------+
| Piano |                                   |
| Keys  |   Canvas (scrollable)             |
| Side  |                                   |
| bar   |                                   |
+------+-----------------------------------+
| Scrollbar (horizontal)                    |
+------------------------------------------+
```

Toolbar row:
- Undo button (disabled when !canUndo): calls editStore.undo()
- Redo button (disabled when !canRedo): calls editStore.redo()
- Text indicator: "Click to add | Right-click to delete | Drag to move"
- Use simple inline SVG icons for undo (curved arrow left) and redo (curved arrow right)

Scrolling:
- Use onWheel handler on the container div
- Horizontal scroll: wheel deltaX (or shift+wheel deltaY)
- Vertical scroll: wheel deltaY (without shift)
- Clamp scrollX: 0 to max(0, totalWidth - viewportWidth)
- Clamp scrollY: 0 to max(0, totalHeight - viewportHeight)
- totalHeight = 128 * NOTE_HEIGHT (full MIDI range)
- totalWidth = (durationTicks + ppq * 4) * pixelsPerTick * zoomX (add 4 beats padding)

Default scroll position: center on C4 (MIDI 60) vertically. Set initial scrollY so that MIDI 60 is near the center of the viewport.

Pass all interaction handlers from usePianoRollInteractions to PianoRollCanvas event props.

Container: `w-full` with a fixed height of 400px (expandable in future). Use `overflow-hidden` and `relative` positioning. Add a border and rounded corners.

**index.ts:**
Barrel export: `export { PianoRollEditor } from './PianoRollEditor';`

Use 'use client' directive on PianoRollEditor.tsx.
