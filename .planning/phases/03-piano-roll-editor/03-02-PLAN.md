  <name>Task 1: Create grid utilities for coordinate conversion</name>
  <files>src/components/PianoRoll/gridUtils.ts</files>
  <action>
Create pure utility functions for converting between MIDI coordinates and canvas pixel coordinates.

Constants:
- `NOTE_HEIGHT = 12` pixels per semitone row
- `DEFAULT_PIXELS_PER_TICK = 0.1` (at zoom 1.0, 480 ticks = 48px = roughly one beat width)
- `MIDI_MIN = 0`, `MIDI_MAX = 127` (full MIDI range, but default view shows ~C2-C6 range)
- `PIANO_KEY_WIDTH = 60` pixels for the sidebar

Functions:
- `ticksToX(ticks: number, pixelsPerTick: number, scrollX: number): number` - Convert tick position to canvas X coordinate
- `xToTicks(x: number, pixelsPerTick: number, scrollX: number): number` - Convert canvas X to ticks (for click-to-place). Snap to grid: round to nearest subdivision based on current zoom.
- `midiToY(midi: number, scrollY: number): number` - Convert MIDI note number to canvas Y. Higher MIDI = higher on screen (invert Y axis). Y = (MIDI_MAX - midi) * NOTE_HEIGHT - scrollY
- `yToMidi(y: number, scrollY: number): number` - Convert canvas Y to MIDI note number. Clamp to 0-127.
- `getGridLines(viewportWidth: number, viewportHeight: number, pixelsPerTick: number, scrollX: number, scrollY: number, ppq: number, timeSignature: { numerator: number; denominator: number }): { vertical: GridLine[]; horizontal: GridLine[] }` - Return arrays of grid lines to draw. Vertical lines: bar lines (bold), beat lines (medium), subdivision lines (light) based on zoom level. Horizontal lines: one per semitone, with C notes highlighted.
- `getSnapTicks(ppq: number, pixelsPerTick: number): number` - Determine snap resolution based on zoom. Zoomed out: snap to beats (ppq ticks). Zoomed in: snap to 16th notes (ppq/4 ticks). Very zoomed in: snap to 32nd notes (ppq/8 ticks). Use pixelsPerTick thresholds to determine level.
- `snapToGrid(ticks: number, snapTicks: number): number` - Round ticks to nearest snap value.

Types:
- `GridLine { position: number; type: 'bar' | 'beat' | 'subdivision'; label?: string }`

This file has NO React dependencies - pure math functions only.

**PianoKeysSidebar.tsx:**
Create a React component that renders a vertical piano keyboard sidebar.

Props:
- `scrollY: number` - Current vertical scroll position
- `height: number` - Viewport height
- `zoomY: number` - Vertical zoom factor (default 1.0)

Rendering:
- Use a canvas element (width: PIANO_KEY_WIDTH from gridUtils, height: prop)
- Draw piano keys vertically: white keys for natural notes, black keys for sharps/flats
- Label every C note (C0 through C9) with text
- Black keys are shorter (60% width) and drawn on top
- Keys scroll in sync with the main canvas via scrollY
- Highlight the C rows with a slightly different shade for orientation

**PianoRollCanvas.tsx:**
Create the main piano roll canvas component.

Props:
- `notes: HaydnNote[]` - Notes to render
- `ppq: number` - Pulses per quarter note
- `timeSignature: { numerator: number; denominator: number }`
- `durationTicks: number` - Total track duration in ticks
- `scrollX: number`, `scrollY: number` - Scroll offsets
- `zoomX: number`, `zoomY: number` - Zoom levels (default 1.0)
- `selectedNoteIds?: Set<string>` - IDs of selected notes (highlighted differently)
- `playheadTicks?: number` - Current playback position to draw playhead line
- `onCanvasClick?: (ticks: number, midi: number) => void` - Click handler for adding notes
- `onNoteClick?: (noteIndex: number, event: React.MouseEvent) => void` - Click on existing note
- `onNoteDragStart?: (noteIndex: number, event: React.MouseEvent) => void` - Start dragging a note
- `trackIndex: number` - For generating note IDs

Rendering (use HTML5 Canvas via useRef + useEffect):
1. Clear canvas
2. Draw horizontal grid lines (one per semitone, C notes brighter)
3. Draw vertical grid lines (bars bold, beats medium, subdivisions light per adaptive grid)
4. Draw notes as filled rounded rectangles:
   - X = ticksToX(note.ticks), width = ticksToX(note.ticks + note.durationTicks) - X
   - Y = midiToY(note.midi), height = NOTE_HEIGHT
   - Color: use HSL based on velocity (brighter = louder). Base hue: 210 (blue). Selected notes: hue 40 (gold).
   - Minimum visual width: 3px (so very short notes are still visible)
5. Draw playhead as a vertical red line at playheadTicks position
6. Re-render on any prop change via useEffect dependency array

Mouse event handling:
- Attach mousedown handler to canvas
- On mousedown: calculate ticks and midi from (x, y) using xToTicks/yToMidi
- Hit-test notes: check if click falls within any note's rectangle bounds
- If hit: call onNoteClick with the note index
- If miss: call onCanvasClick with the calculated (ticks, midi)

Performance:
- Use `requestAnimationFrame` for render batching
- Do NOT use React state for canvas drawing - use refs and imperative drawing
- Redraw only when props change (useEffect dependencies)

Use 'use client' directive at top of both component files.
