<task type="auto">
  <name>Task 1: Create HistoryManager utility</name>
  <files>src/state/historyManager.ts</files>
  <action>
Create a generic HistoryManager class that manages undo/redo for any state type T.

Interface:
- `constructor(initialState: T, maxHistory?: number)` - maxHistory defaults to 100
- `push(state: T): void` - Push new state, clear redo stack
- `undo(): T | null` - Return previous state or null if at beginning
- `redo(): T | null` - Return next state or null if at end
- `canUndo(): boolean`
- `canRedo(): boolean`
- `getCurrent(): T` - Return current state
- `clear(initialState: T): void` - Reset history with new initial state

The state type T should be the track's notes array (`HaydnNote[]`). Each push creates a snapshot.

Use deep clone via `structuredClone()` (available in all modern browsers) to ensure immutability of history entries. Do NOT use JSON.parse/JSON.stringify - it's slower and loses undefined values.

Cap history at maxHistory entries to prevent memory bloat on long editing sessions.
  </action>
  <verify>TypeScript compiles without errors: `npx tsc --noEmit src/state/historyManager.ts`</verify>
  <done>HistoryManager class exports with push/undo/redo/canUndo/canRedo/getCurrent/clear methods. Uses structuredClone for immutability. Caps at 100 entries by default.</done>
</task>

<task type="auto">
  <name>Task 2: Create edit store with note CRUD and undo/redo</name>
  <files>src/state/editStore.ts</files>
  <action>
Create a Zustand store (`useEditStore`) that manages all editing state for the piano roll.

State:
- `selectedTrackIndex: number | null` - Currently selected track for editing
- `selectedNoteIds: Set<string>` - Selected note indices (for future multi-select; use `${trackIndex}-${noteIndex}` as ID)
- `canUndo: boolean`
- `canRedo: boolean`
- `isEditing: boolean` - True when any drag/edit operation is in progress

Actions:
- `selectTrack(index: number): void` - Set selected track, initialize history for that track's notes
- `addNote(note: HaydnNote): void` - Add note to selected track, push history, sync to projectStore
- `deleteNote(noteIndex: number): void` - Remove note by index, push history, sync to projectStore
- `moveNote(noteIndex: number, newMidi: number, newTicks: number): void` - Update note position, push history, sync to projectStore
- `updateNote(noteIndex: number, updates: Partial<HaydnNote>): void` - Update any note properties, push history, sync to projectStore
- `undo(): void` - Restore previous state from history, sync to projectStore
- `redo(): void` - Restore next state from history, sync to projectStore
- `selectNote(noteId: string): void` - Select a single note
- `clearSelection(): void` - Deselect all notes
- `setEditing(editing: boolean): void`

Internal:
- Maintain a `HistoryManager<HaydnNote[]>` instance per track (create on selectTrack, reset on track change)
- When syncing to projectStore: use `useProjectStore.getState()` to get current project, clone the tracks array, replace the edited track's notes, call `setProject()` with updated project

Important patterns:
- Follow existing Zustand patterns from projectStore.ts and playbackStore.ts
- Use `create` from 'zustand' (no middleware needed for v1)
- Access projectStore via `useProjectStore.getState()` (cross-store communication pattern)
- Notes are sorted by ticks after any mutation (addNote, moveNote)
  </action>
