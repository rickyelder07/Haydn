Create `src/app/api/ai-compose/route.ts`. The route accepts POST requests and dispatches to either `handleClarify` or `handleGenerate` based on the presence of `generateAttempt` in the body.

**Request parsing and validation:**
```typescript
const { prompt, clarifyingHistory = [], generateAttempt, validationErrors } = await request.json();
if (!prompt || typeof prompt !== 'string' || prompt.length === 0) {
  return NextResponse.json({ error: 'Missing or invalid prompt' }, { status: 400 });
}
if (prompt.length > 500) {
  return NextResponse.json({ error: 'Prompt exceeds 500 character limit' }, { status: 400 });
}
```

**Dispatch logic:**
```typescript
if (generateAttempt !== undefined) {
  return handleGenerate(prompt, clarifyingHistory, generateAttempt, validationErrors);
} else {
  return handleClarify(prompt, clarifyingHistory);
}
```

```
You are a music composition assistant. A user wants you to compose original MIDI music.
Ask up to 3 clarifying questions to better understand their musical vision.

Ask ONLY questions that will meaningfully change the composition:
- Mood or emotion (e.g., melancholic, energetic, peaceful)
- Length (e.g., 16 bars, 32 bars)
- Specific instruments or style elements

Rules:
- Ask at most 3 questions
- Ask all questions at once (not one at a time)
- If the user's prompt is already detailed, return an empty questions array
- Keep questions concise and non-technical
```

Messages array:
```typescript

  { role: 'system', content: clarifySystemPrompt },
  { role: 'user', content: prompt },
  ...history,  // any prior clarify turns (usually empty on first call)
]
```

Use `zodResponseFormat(QuestionsSchema, 'questions')` for the response_format.

Return: `{ type: 'questions', data: parsed, usage }` — where `parsed` is `{ questions: string[] }`.

**handleGenerate(prompt, history, attempt, validationErrors):**

System prompt (build as a function):
```
You are a MIDI composer. Generate a complete, original MIDI composition as structured JSON.

## Timing Rules (CRITICAL)
PPQ (Pulses Per Quarter Note) = 480
- Beat 1 of bar 1 = tick 0
- Each beat = 480 ticks
- Bar length (4/4) = 4 beats × 480 = 1920 ticks
- Bar 2 beat 1 = 1920, bar 3 beat 1 = 3840, bar 3 beat 2 = 3840 + 480 = 4320
- Align ALL note start times to beat grid (multiples of 480)

## Composition Rules
- Generate 16-32 bars maximum
- Use multiple tracks: drums on channel 9, melodic tracks on channels 0-3
- Velocity: 0.0-1.0 range (typical notes: 0.6-0.85, accents: 0.9)
- Do NOT overlap notes on the same pitch in the same track
- Drums use channel 9 (GM standard percussion map: kick=36, snare=38, hihat=42/44/46)

## Quality Standards
- Produce musical, rhythmically coherent output
- Bass notes should mostly align with chord root notes
- Melodies should move mostly by steps with occasional leaps
- Include dynamics: vary velocity across notes (not all identical)
```

Messages array:
```typescript
[
  { role: 'system', content: generateSystemPrompt },
  { role: 'user', content: prompt },
  ...history,
  // If attempt > 0 and validationErrors exist, append a corrective message:
  ...(attempt > 0 && validationErrors?.length
    ? [{ role: 'user', content: `Previous attempt failed music theory validation:\n${validationErrors.join('\n')}\nPlease fix these issues in the new composition.` }]
    : [])
]
```

Use `zodResponseFormat(MidiCompositionSchema, 'midi_composition')` for the response_format. Model: `'gpt-4o-2024-08-06'`.

Return: `{ type: 'composition', data: parsed, usage }`.

**Error handling:** Wrap each handler in try/catch. On any error return `{ error: string }` with status 500. Log errors with `console.error('[ai-compose]', error)`.
