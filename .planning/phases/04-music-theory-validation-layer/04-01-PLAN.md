    2. Create `src/lib/music-theory/types.ts` with these types:

    ```typescript
    // Validation error severity
    type ValidationSeverity = 'error' | 'warning';

    // Individual validation error
    interface ValidationError {
      type: 'scale' | 'progression' | 'transition' | 'genre';
      severity: ValidationSeverity;
      message: string;              // User-friendly (e.g., "F# is not in C major scale")
      technicalDetail: string;      // Dev info (e.g., "MIDI 66 pitch class 6 not in scale [0,2,4,5,7,9,11]")
      affectedMidi?: number;        // The MIDI note that caused the issue
    }

    // Discriminated union Result type (no library needed)
    type ValidationResult =
      | { ok: true }
      | { ok: false; errors: ValidationError[] };

    // Context passed to every validator
    interface ValidationContext {
      note: HaydnNote;               // The note being validated
      track: HaydnTrack;             // The track it belongs to
      project: HaydnProject;         // Full project for metadata access
      editType: 'add' | 'move' | 'update';
    }

    // Validator interface -- all validators implement this
    interface Validator {
      name: string;
      validate(context: ValidationContext): ValidationResult;
    }

    // Genre rule configuration (data, not code)
    interface GenreRules {
      name: string;
      displayName: string;
      allowedScales: string[];           // Tonal scale names (e.g., "minor", "harmonic minor")
      preferredIntervals: number[];      // Semitone intervals typical for genre
      avoidedIntervals: number[];        // Semitone intervals unusual for genre
      allowedChordTypes: string[];       // Tonal chord types
      strictness: 'strict' | 'moderate' | 'permissive';
    }

    // Note conformance info for visual feedback
    interface NoteConformance {
      midi: number;
      inScale: boolean;
      scaleName?: string;
    }
    ```

    Export all types. Import HaydnNote, HaydnTrack, HaydnProject, HaydnKeySignature from '@/lib/midi/types'.

    IMPORTANT: Use `export type` for interfaces/types and `export` for any runtime values. Keep this file pure types (no runtime code).
  </action>

    1. Create `src/lib/music-theory/pitchClass.ts`:

    Import `Note` and `Scale` from 'tonal'.

    Functions to implement:
    - `midiToPitchClass(midi: number): number` -- returns midi % 12 (0-11)
    - `midiToNoteName(midi: number): string` -- uses Note.fromMidi(midi), returns e.g. "C4", "F#5"
    - `midiToPitchClassName(midi: number): string` -- uses Note.pitchClass(Note.fromMidi(midi)), returns e.g. "C", "F#"
    - `isNoteInScale(midiNote: number, scaleName: string): boolean` -- uses Scale.get(scaleName) to get scale notes, compares using Note.chroma() for enharmonic-safe comparison. Return true if pitch class matches any scale note's chroma. If scale is invalid/empty, return true (permissive -- don't block on bad data).
    - `getScaleNotes(scaleName: string): number[]` -- returns array of pitch class numbers (0-11) in the scale. Uses Scale.get() and Note.chroma(). Returns empty array for invalid scale.
    - `getScalePitchClasses(key: string, scale: string): Set<number>` -- convenience that builds `${key} ${scale}` and returns Set of chroma values for fast lookup.

    CRITICAL: Use Note.chroma() for comparison, NEVER string comparison on note names. This avoids the enharmonic equivalence pitfall (F# vs Gb).

    2. Create `src/lib/music-theory/keySignature.ts`:

    Import HaydnKeySignature from '@/lib/midi/types'.
    Import { Scale, Note } from 'tonal'.
    Import { getScalePitchClasses } from './pitchClass'.

    Functions to implement:
    - `getCurrentKeySignature(keySignatures: HaydnKeySignature[], ticks: number): HaydnKeySignature | null` -- finds most recent key signature at or before given tick. Sort by ticks descending, return first where ks.ticks <= ticks. Return null if no key signatures exist.
    - `getScaleAtTick(keySignatures: HaydnKeySignature[], ticks: number): { scaleName: string; pitchClasses: Set<number> } | null` -- combines getCurrentKeySignature with getScalePitchClasses. Returns null if no key signature at that position. scaleName format: "${key} ${scale}" (e.g., "C major", "A minor").

    Handle edge case: If keySignatures array is empty, return null (project is chromatic/atonal, validation should be permissive).
