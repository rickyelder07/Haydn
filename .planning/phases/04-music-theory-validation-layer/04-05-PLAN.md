<task type="checkpoint:human-verify" gate="blocking">
  <what-built>
    Complete Music Theory Validation Layer including:
    - ScaleValidator: rejects notes outside current key signature's scale
    - TransitionValidator: warns about notes that clash with surrounding musical context
    - GenreValidator: warns about genre-atypical intervals
    - ValidationPipeline: coordinates all validators
    - Visual feedback: piano roll row highlighting for scale conformance
    - Validation errors/warnings displayed as banners
    - Theory controls: validation toggle, genre selector, key display
  </what-built>
  <how-to-verify>
    Start the dev server: `npm run dev`
    Open http://localhost:3000

    **Test 1: Scale Validation (Success Criterion 1)**
    1. Upload a MIDI file that has a key signature (or any MIDI file)
    2. Select a track in the piano roll
    3. If the file has a key signature, try adding a note OUTSIDE the scale (e.g., if in C major, click to place a note on F# row)
    4. Expected: Red error banner appears with message like "F# is not in C major scale". The note should NOT be added.
    5. Then add a note INSIDE the scale (e.g., on C, D, E, F, G, A, or B row in C major)
    6. Expected: Note is added successfully, no error banner.

    **Test 2: Visual Feedback (Success Criterion 4)**
    1. With validation enabled, observe the piano roll grid
    2. Expected: In-scale rows have a subtle blue tint, out-of-scale rows have a faint red tint
    3. Toggle validation off using the Theory toggle in the toolbar
    4. Expected: Row highlighting disappears

    **Test 3: Genre Selector (Success Criterion 2)**
    1. Select "Classical" from the genre dropdown
    2. Add a note that creates a tritone interval with the previous note
    3. Expected: Amber warning banner about unusual interval for classical style
    4. Switch to "Jazz" genre
    5. Try the same interval
    6. Expected: No warning (tritone is acceptable in jazz)

    **Test 4: Validation Toggle**
    1. Turn validation OFF using the toggle
    2. Try adding a note outside the scale
    3. Expected: Note is added (no validation blocking)
    4. Turn validation back ON
    5. Expected: Validation resumes

    **Test 5: Transition Warning (Success Criterion 3)**
    1. Place several notes in a clear key (e.g., C, E, G = C major triad)
    2. Try adding a note nearby that's outside the detected context (e.g., Db)
    3. Expected: Amber warning about jarring transition (but note is still added since it's a warning, not error)

    **Test 6: Edge Cases**
    1. Upload a MIDI file with NO key signature
    2. Expected: Scale validation is permissive (all notes allowed), row highlighting absent
    3. Current key display shows "Key: --"
  </how-to-verify>
  <resume-signal>Type "approved" if all tests pass, or describe any issues found</resume-signal>
</task>
