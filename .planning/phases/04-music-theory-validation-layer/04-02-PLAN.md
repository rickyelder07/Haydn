  <name>ScaleValidator</name>
  <files>
    src/lib/music-theory/validators/ScaleValidator.ts
    src/lib/music-theory/validators/__tests__/ScaleValidator.test.ts
  </files>
  <behavior>
    ScaleValidator implements the Validator interface. It checks whether a note's pitch class belongs to the current key signature's scale at the note's tick position.

    Test cases (RED phase):

    1. C4 (MIDI 60) in C major -> ok: true (C is in C major)
    2. F#4 (MIDI 66) in C major -> ok: false, error message contains "F#" and "C major"
    3. Bb3 (MIDI 58) in F major -> ok: true (Bb is in F major -- enharmonic test)
    4. C#5 (MIDI 73) in D major -> ok: true (C# is in D major)
    5. No key signature in project -> ok: true (permissive default)
    6. Key change mid-song: note at tick 960 with key change from C to G at tick 480 -> validates against G major (not C)
    7. Invalid/unknown scale -> ok: true (permissive, don't block)
    8. Error severity should be 'error' (not 'warning')
    9. Error type should be 'scale'
    10. Error message should be user-friendly: "${noteName} is not in ${scaleName} scale"
  </behavior>
  <implementation>
    After tests pass, ScaleValidator should:
    1. Call getCurrentKeySignature(project.metadata.keySignatures, note.ticks)
    2. If no key signature, return { ok: true }
    3. Build scale name: `${key} ${scale}`
    4. Get scale via Scale.get(scaleName)
    5. If scale invalid/empty notes, return { ok: true }
    6. Get note's pitch class via Note.chroma(Note.pitchClass(Note.fromMidi(note.midi)))
    7. Check if pitch class is in scale notes using Note.chroma() comparison
    8. If not in scale, return error with user-friendly message
  </implementation>
</feature>

<feature>
  <name>TransitionValidator</name>
  <files>
    src/lib/music-theory/validators/TransitionValidator.ts
    src/lib/music-theory/validators/__tests__/TransitionValidator.test.ts
  </files>
  <behavior>
    TransitionValidator checks whether a new/moved note fits the musical context of surrounding notes. It detects scales from nearby notes and warns if the new note doesn't fit.

    Test cases (RED phase):

    1. Note added near C major context (C, D, E, F, G notes nearby) with C4 -> ok: true
    2. Note added near C major context with F#4 -> ok: false, warning about transition
    3. No surrounding notes (empty track or isolated) -> ok: true
    4. Few surrounding notes (< 3) -> ok: true (not enough context to detect scale)
    5. Severity should be 'warning' (not 'error' -- transitions are suggestions)
    6. Error type should be 'transition'
    7. Neighborhood window: notes within +/- 2 beats (ppq * 2 ticks) of the note

    Edge cases:
    8. Note being validated should be excluded from neighborhood detection
    9. Notes with same tick position as target should be included in context
  </behavior>
  <implementation>
    After tests pass, TransitionValidator should:
    1. Get notes within time window (note.ticks +/- ppq*2) from track.notes
    2. Exclude the note being validated itself
    3. If fewer than 3 neighboring notes, return { ok: true } (insufficient context)
    4. Convert neighboring MIDI notes to note names using Note.fromMidi()
    5. Use Scale.detect() to find possible scales
    6. If no scales detected, return { ok: true }
    7. Check if new note fits any detected scale (using Note.chroma())
    8. If doesn't fit, return warning with detected scale context
  </implementation>
</feature>
