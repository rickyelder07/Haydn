  <action>
    1. Create `src/lib/music-theory/rules/genres.ts`:

    Define genre presets as plain data objects conforming to GenreRules type from types.ts. Include at minimum:

    - CLASSICAL: allowedScales = ["major", "minor", "harmonic minor"], strict, avoids tritone (6) in melody
    - JAZZ: allowedScales = ["major", "dorian", "mixolydian", "lydian", "altered"], permissive, prefers 3rds/7ths (intervals 3,4,10,11)
    - TRAP: allowedScales = ["minor", "harmonic minor", "phrygian"], moderate, prefers minor intervals (1,3,6)
    - POP: allowedScales = ["major", "minor", "mixolydian"], moderate, prefers consonant intervals (3,4,5,7)
    - NONE: empty arrays, permissive (no genre validation)

    Export as `GENRE_PRESETS: Record<GenrePresetKey, GenreRules>` where GenrePresetKey = 'classical' | 'jazz' | 'trap' | 'pop' | 'none'.

    2. Create `src/lib/music-theory/validators/GenreValidator.ts`:

    Implements Validator interface. Constructor takes GenreRules.

    validate() logic:
    - Find previous note in track (note with ticks < context.note.ticks, closest to it)
    - If no previous note, return { ok: true }
    - Calculate interval: Math.abs(context.note.midi - previousNote.midi) % 12
    - If interval is in rules.avoidedIntervals, return warning (severity: 'warning', type: 'genre')
    - Message: "Interval of ${intervalName} sounds unusual for ${rules.displayName} style"
    - Map interval numbers to names: 0=unison, 1=minor 2nd, 2=major 2nd, 3=minor 3rd, etc.
    - If strictness is 'permissive', always return { ok: true } (skip validation)

    3. Create `src/lib/music-theory/validators/ValidationPipeline.ts`:

    Class ValidationPipeline:
    - `validators: Validator[]` private array
    - `addValidator(v: Validator): this` -- chain method
    - `validate(context: ValidationContext): ValidationResult` -- runs all validators, collects ALL errors and warnings (does not short-circuit). Returns { ok: false, errors: [...] } if any error-severity issues. Returns { ok: true } with warnings stored separately if only warnings.

    Actually, revise the return type to include warnings even on success:
    ```typescript
    interface PipelineResult {
      valid: boolean;         // true if no 'error' severity issues
      errors: ValidationError[];   // severity: 'error' items
      warnings: ValidationError[]; // severity: 'warning' items
    }
    ```

    Add this PipelineResult type to types.ts.

    Export factory function `createDefaultPipeline(genreRules?: GenreRules): ValidationPipeline` that wires up ScaleValidator + TransitionValidator + GenreValidator (if genreRules provided).
  </action>

  <action>
    1. Create `src/state/validationStore.ts` (Zustand store):

    State:
    - `enabled: boolean` (default: true -- validation on by default)
    - `activeGenre: GenrePresetKey` (default: 'none' -- no genre constraint initially)
    - `lastErrors: ValidationError[]` (errors from last validation)
    - `lastWarnings: ValidationError[]` (warnings from last validation)
    - `scalePitchClasses: Set<number> | null` (current scale's pitch classes for visual feedback, null if no key sig)
    - `currentScaleName: string | null` (display name like "C major")

    Actions:
    - `setEnabled(enabled: boolean)` -- toggle validation
    - `setGenre(genre: GenrePresetKey)` -- switch genre preset
    - `setLastResult(errors: ValidationError[], warnings: ValidationError[])` -- update after validation
    - `clearResult()` -- clear errors/warnings
    - `updateScaleInfo(keySignatures: HaydnKeySignature[], ticks: number)` -- compute and cache current scale pitch classes for visual feedback using getScaleAtTick()

    2. Modify `src/state/editStore.ts`:

    Add a private helper function `validateNote(note: HaydnNote, editType: 'add' | 'move' | 'update'): boolean`:
    - Import useValidationStore and useProjectStore
    - Read validation state: enabled, activeGenre
    - If not enabled, return true (skip validation)
    - Build ValidationContext from note + current track + project
    - Create pipeline via createDefaultPipeline() with genre rules from GENRE_PRESETS[activeGenre]
    - Run pipeline.validate(context)
    - Store result in validationStore (setLastResult)
    - Return result.valid (true = allow edit, false = block edit)

    Modify addNote():
    - Before pushing to history, call validateNote(note, 'add')
    - If returns false, do NOT apply the edit (early return)
    - If returns true, proceed as before

    Modify moveNote():
    - Before pushing to history, call validateNote with updated note, 'move'
    - If returns false, do NOT apply the move
    - If returns true, proceed

    Modify updateNote():
    - Only validate if midi or ticks changed (not for velocity-only changes)
    - Before pushing to history, call validateNote, 'update'
    - If returns false, do NOT apply update

    IMPORTANT: Do NOT validate on undo/redo -- those restore known-good states.
    IMPORTANT: Read stores via getState() pattern (not hooks) since this is outside React.
    IMPORTANT: Clear validation result (clearResult) at the start of each edit attempt.
  </action>
